# Privacy Policy — ccgauge-bar

**Effective date**: 2026-05-24

ccgauge-bar is designed around the principle that your AI-tool usage
data should never leave your machine without an explicit opt-in. This
document is the authoritative statement of what data the app reads,
where it goes, and what it doesn't do.

## TL;DR

- ccgauge-bar reads two folders on your local disk:
  `~/.claude/projects/**/*.jsonl` and `~/.codex/sessions/**/*.jsonl`.
- It computes token usage and cost from those files **entirely on
  your machine**.
- It makes **zero outbound network requests by default**. Every
  outbound traffic is **user-triggered** and falls into one of the
  rows in [Network behaviour](#network-behaviour) below: clicking
  **检查更新**, clicking **详情↗** / **Open Dashboard**, or clicking
  any of the **GitHub / Issues / Privacy** links in Settings → About.
- It collects **no telemetry**, no analytics, no crash reports.
- It writes nothing back to your Claude / Codex JSONL files.
- It keeps a cache of file metadata **plus parsed records** (each
  record retains up to 200 characters of the corresponding message
  text — same as the in-app preview) at
  `~/Library/Application Support/CCGaugeBar/cache/index-v1.json` so
  warm launches are fast. That cache **stays on your machine**; you
  can delete the folder at any time and the app will rebuild on next
  launch.

## What data is read

ccgauge-bar reads the JSONL session logs that **you** generated by
running `claude code` and `codex` locally. Specifically:

| Folder | Owner | What's in it |
|---|---|---|
| `~/.claude/projects/` (or `~/.config/claude/projects/`) | Claude Code | One JSONL file per conversation: user prompts, assistant responses, tool calls, token usage |
| `~/.codex/sessions/` and `~/.codex/archived_sessions/` | OpenAI Codex CLI | Per-session event streams: user / agent messages, tool calls, token counts |

The app extracts numeric fields (tokens in / out / cache, timestamps,
model names, project paths) and the first 200 characters of each
user / assistant message (for display in the in-app usage table —
**this never leaves your machine**).

If `CCGAUGE_CONFIG_DIR`, `CLAUDE_CONFIG_DIR`, `CCGAUGE_CODEX_DIR`, or
`CODEX_HOME` environment variables are set, the app also scans the
corresponding paths.

## What data is NOT read

- No clipboard contents
- No browser history
- No keystroke / mouse / window event monitoring
- No microphone / camera / location
- No iCloud / Apple ID data
- No SSH / Git credentials (the app never opens `.ssh/` or
  `.gitconfig`)

## Network behaviour

| When | What is contacted | Why |
|---|---|---|
| **Always** (default) | Nothing | The whole feature set works offline |
| You click **Settings → About → 检查更新** | `https://api.github.com/repos/chengzuopeng/ccgauge-app/releases/latest` | Compare your installed `CFBundleShortVersionString` to the latest published GitHub release tag, so the app can tell you a newer version exists |
| You click **详情↗** in the header or **Open Dashboard** in the right-click menu — **step 1** | `http://localhost:3737` (localhost only) | Probe whether the `ccgauge` npm dashboard is running locally. If reachable, open it in your default browser and stop here |
| You click **详情↗** / **Open Dashboard** and the localhost probe above fails — **step 2** | `https://chengzuopeng.github.io/ccgauge/` | Fall back to the project's public landing page so the click still has somewhere to go. This is a normal browser navigation; the app itself does not transmit any data with it |
| You click **GitHub** / **Issues** / **Privacy** in Settings → About | `https://github.com/chengzuopeng/ccgauge-app[/issues / /blob/main/PRIVACY.md]` | Hand off to your default browser to open the project's repository, the issue tracker, or this very document. The app passes only the static URL; nothing about you or your data is sent |

The app does NOT, and will not without an explicit code-visible
change, contact any other endpoint. In particular: nothing is sent
to any analytics, advertising, crash-reporting, or telemetry
service.

## Where data is stored

| Location | Contents | Lifetime |
|---|---|---|
| `~/Library/Application Support/CCGaugeBar/cache/index-v1.json` | Per-file mtime + size for change detection, plus the parsed records (model names, token counts, timestamps, tool names, and the first ≤200 characters of each user / assistant message — same fields the in-app usage table displays) | Deleted by removing the folder; the app rebuilds on next launch |
| `~/Library/Preferences/dev.ccgauge.bar.plist` | Your settings (source, range, sort, language, currency) | Managed by macOS `NSUserDefaults` |

Beyond those two paths, the app writes to disk only when you
explicitly **export a CSV** from the Usage page — the macOS save panel
lets you pick any location you want, and the app writes the exact rows
shown in that table at that moment. ccgauge-bar never writes anywhere
else on disk without your explicit click.
